mysql 事务

发表于 2025-02-07 分类于 mysql

手动提交事务

通过 set autocommit = 0 | false; 关闭事务自动提交，然后手动执行 commit 或者 rollback。
语法：

注意：
关闭自动提交后，直到执行 COMMIT 或 ROLLBACK 前，所有的 SQL 语句都会被视为一个事务的一部分。
commit 提交事务，将所有更改永久保存到数据库中。
rollback 回滚事务，撤销自上次 commit 或 rollback 后执行的所有更改。
设置 set autocommit = 1 | true; 可以重新开启自动提交。
通过 set autocommit 设置事务的提交方式，只适用于当前会话。
通过 show variables like ‘autocommit’; 可以查看当前会话的自动提交状态。

开启独立事务

通过 start transaction; 开启一个独立的事务，然后执行一系列的 SQL 语句，最后通过 COMMIT 或 ROLLBACK 来提交或回滚事务。
语法：

注意：
不管是否关闭自动提交，都可以通过 start transaction 开启一个独立的事务。

事务特性

事务不支持 DDL 语句（如 CREATE TABLE、ALTER TABLE 等），因为这些操作会立即生效，无法回滚。

事务隔离

隔离作用：
避免并发事务之间的干扰，保证数据的一致性。
隔离级别：
read-uncommitted（读未提交）：最低的隔离级别，会出现 脏读 不可能重复读 幻读。
read-committed（读已提交）：避免了脏读，可能出现 不可重复读 和幻读。
repeatable-read（可重复读）：避免了脏读和不可重复读，可能出现幻读。
serializable（可串行化）：最高的隔离级别，避免了脏读、不可重复读和幻读。
隔离问题：
脏读：一个事务读取了另一个未提交的事务的数据（真实错误）。
不可重复读：一个事物读取了另一个事务提交的修改数据（不符合一致性原则，不是数据错误）。
幻读：一个事务读取了另一个事务提交的插入数据（不符合一致性原则，不是数据错误）。
修改查看语法：
select @@transaction_isolation;
set transaction_isolation = 隔离级别;
总结：
隔离级别越高，数据越安全，性能越低。建议设置隔离级别为 read-committed。

mysql 多表查询

发表于 2025-02-07 分类于 mysql

表关系

一对一
特点：双方的数据行都对应对方至多一条数据
意义：用于优化冷热数据查询
如何建立关系：
方式一：子表的外键与主键融合，两张表公用同一个主键。
方式二：子表外键添加唯一约束

一对多
特点：主表对应多条子表数据，子表对应主表至多一条数据
意义：解决数据冗余问题。
如何建立关系：
子表的外键不进行约束。

多对多
特点：双方的数据行都可以对应对方多条数据
如何建立关系：
多对多需要创建中间表建立数据之间的关联。
中间表包含两个外键，主表数据之间间接关联。

多表查询

内连接（inner join）

作用：
可以将多个表的数据行，正确水平拼接到一个结果集中。

存在N个表相连接时，会先将前两个表进行连接，再将结果与第三个表进行连接。

嵌套子查询

概念：一个sql语句中嵌套了另一个或多个完整的sql语句，称为子查询或内查询。

根据子查询的结果，可分为4种情况：
标量子查询：子查询返回结果是一行一列，单一值，一般用在条件判断。
行子子查询：子查询返回结果是一行多列，一般用于插入数据的值，或者整体对比。
列子子查询：子查询返回结果是一列多行，一般用于条件对比，需要配合 in any all 等关键字。
表子子查询：子查询返回结果是多行多列，不能用于条件判断，一般用于查询的虚拟表的中间表。

提示：
any all 关键字只用在列子查询中。
any 关键字相当于 or 关系，all 关键字相当于 and 关系。

子查询可以在select，update，delete，insert，等语句中使用。

mysql 约束

发表于 2025-02-06 分类于 mysql

NOT NULL

作用：确保列中不能有NULL值。

DEFAULT

作用：为列提供一个默认值。

提示：如果主动插入NULL值，则不会触发默认值行为。

CKECK

作用：确保列中的值满足某个条件。

AUTO_INCREMENT

作用：自动增长约束，确保列中的数据唯一且不为空。

提示：
添加AUTO_INCREMENT约束的列必须是主键或者唯一约束。
每个表只能有一个AUTO_INCREMENT约束。
自增列必须是整数类型。
插入数据时，不指定自增列的值或值为null或0时，会自动增长。

PIRMARY KEY

作用：主键约束，确保列中的数据唯一且不为空。

提示：一个表只能有一个主键，但是可以有多个字段组合成一个复合主键。

UNIQUE

作用：唯一约束，确保列中的数据唯一。

提示：同一个表上可以多个字段有唯一约束，或者多个字段组合唯一。

FOREIGN KEY

作用：外键约束，确保列中的数据在另一个表中存在。

外键概念：外键就是在一个表中定义另一个表的主键列，从而达到两个表之间的关联。
外键约束概念：外键约束是用来保证外键数据正确的限制。外键没有外键约束也是外键，最好添加外键约束。
级联配置：
在定义外键时，可以使用ON DELETE 配置 ON UPDATE 配置 关键字来指定当父表数据被删除或更新时，子表应该执行的操作。
CASCADE 级联删除/更新，如果删除/更新父表的记录，子表对应记录也被删除/更新。
SET NULL 子表的外键允许为空，且删除父表记录时，子表数据应保留但失去关联，如部门被删除时，员工仍然存在，但 department_id 变 NULL。
RESTRICT / NO ACTION（默认）限制删除/更新父表记录，如果子表有对应数据则不允许操作。

注意：
一个表可以有多个外键约束。
外键是跨表引用其他表的主键值。
外键所在的表叫子表，被引用的主键所在的表叫主表。
外键的类型必须必须和主键的类型相同，命名最好相同。
主外键关系，就代表两张表存在关联关系，之后可以进行多表的水平查询。
有主外键约束，删除主表数据的前提是子表没有引用数据！否则应该先删除子表的引用数据，再删除主表数据！

补充

注意：
使用ALTER TABLE 表名 MODIFY 字段 数据类型;进行修改时，会将之前的 NOT NULL、DEFAULT、AUTO_INCREMENT约束一并移除。

mysql 高级查询

发表于 2025-02-04 分类于 mysql

分组查询

分组是将数据分成若干个组，并对每个组进行单独的查询，最终统计的是组的特性和数据（分组列，聚合函数）。
语法：
group by 分组列, 分组列, … hanving 分组后进行比较。
提示：
分组查询只能查询分组字段和聚合函数。
having是分组后的条件，where是分组前的条件。
having只能出现在group by后面，where可以出现在任何位置。
having通常比较的是聚合函数，where可以是任何参数比较。
having可以引用别名，where不可以，因为having是在分组之后执行，已经有了查询结果。

排序查询

语法：
order by 列名 asc/desc, 列名 asc/desc, …
提示：
asc升序 从小到大，desc降序 从大到小。
默认是asc，可以不写。
order by后面可以有多个列，按照第一个排序，如果相同则按第二个排序，以此类推。

分页查询

语法：
limit 起始索引, 每页条数
提示：
起始索引可以省略，省略默认为0。
limit影响的是查询结果，通常用在查询的最后。

查询的执行流程

关键字书写的顺序：
select -> from -> where -> group by -> having -> order by -> limit
关键字的执行顺序：
from -> where -> group by -> having -> select -> order by -> limit
提示：
mysql8.0+版本存在优化，使 group by 、having 可以使用 select 产生的列（别名）。

等于对比：= 不能做空判断 null = null 结果为 false，null <=> null 结果为 true（mysql独有）

区间比较：between min and max not between min and max
范围比较：key in(x,y,z) key = x or key = y or key = z not in
模糊匹配：like not like like 匹配规则，%任意字符出现0次或多次，_单一字符。

逻辑运算符

语法：and or not xor
注意：数据库中非空 非0代表true 反之为false, true -> 1 | false -> 0

运算符优先级

优先级 运算符 说明
1（最高） ! 逻辑非
2 *，/，div，%，mod 乘、除、整除、取模，取模
3 +，- 加、减
4 =，<=>，>=，>，<，<=，!=，IS，LIKE，IN，REGEXP 比较运算
5 BETWEEN，CASE，WHEN，THEN，ELSE 逻辑运算
6 NOT 逻辑非
7 AND 逻辑与
8 XOR 逻辑异或
9（最低） OR 逻辑或

注意：
一般情况下的运算符优先级。在实际使用中，如果不确定优先级，可以使用括号来明确运算的顺序！()内优先级最高！
