mysql 约束

发表于 2025-02-06 分类于 mysql

NOT NULL

作用：确保列中不能有NULL值。

DEFAULT

作用：为列提供一个默认值。

提示：如果主动插入NULL值，则不会触发默认值行为。

CKECK

作用：确保列中的值满足某个条件。

AUTO_INCREMENT

作用：自动增长约束，确保列中的数据唯一且不为空。

提示：
添加AUTO_INCREMENT约束的列必须是主键或者唯一约束。
每个表只能有一个AUTO_INCREMENT约束。
自增列必须是整数类型。
插入数据时，不指定自增列的值或值为null或0时，会自动增长。

PIRMARY KEY

作用：主键约束，确保列中的数据唯一且不为空。

提示：一个表只能有一个主键，但是可以有多个字段组合成一个复合主键。

UNIQUE

作用：唯一约束，确保列中的数据唯一。

提示：同一个表上可以多个字段有唯一约束，或者多个字段组合唯一。

FOREIGN KEY

作用：外键约束，确保列中的数据在另一个表中存在。

外键概念：外键就是在一个表中定义另一个表的主键列，从而达到两个表之间的关联。
外键约束概念：外键约束是用来保证外键数据正确的限制。外键没有外键约束也是外键，最好添加外键约束。
级联配置：
在定义外键时，可以使用ON DELETE 配置 ON UPDATE 配置 关键字来指定当父表数据被删除或更新时，子表应该执行的操作。
CASCADE 级联删除/更新，如果删除/更新父表的记录，子表对应记录也被删除/更新。
SET NULL 子表的外键允许为空，且删除父表记录时，子表数据应保留但失去关联，如部门被删除时，员工仍然存在，但 department_id 变 NULL。
RESTRICT / NO ACTION（默认）限制删除/更新父表记录，如果子表有对应数据则不允许操作。

注意：
一个表可以有多个外键约束。
外键是跨表引用其他表的主键值。
外键所在的表叫子表，被引用的主键所在的表叫主表。
外键的类型必须必须和主键的类型相同，命名最好相同。
主外键关系，就代表两张表存在关联关系，之后可以进行多表的水平查询。
有主外键约束，删除主表数据的前提是子表没有引用数据！否则应该先删除子表的引用数据，再删除主表数据！

补充

注意：
使用ALTER TABLE 表名 MODIFY 字段 数据类型;进行修改时，会将之前的 NOT NULL、DEFAULT、AUTO_INCREMENT约束一并移除。

mysql 函数

常规函数

数字函数
abs(x): 返回 x 的绝对值
ceil(x): 返回 大于或等于 x 的最小整数（向上取整）
floor(x): 返回 小于或等于 x 的最大整数（向下取整）
round(x): 返回 x 四舍五入最接近的整数
round(x,n): 返回 x 四舍五入保留 n 位小数
rand(): 返回 0 到 1 之间的随机浮点数（0 ≤ x < 1）
rand(x): 当 x 相同时，RAND(x) 返回相同的随机数（用于伪随机种子）
truncate(x,n): 返回 x 截断到 n 位小数（不四舍五入）

字符串函数
concat(str1, str2, …): 返回连接参数产生的字符串
char_length(str): 返回 str 的字符数
find_in_set(str, strlist): 返回 str 在字符串列表 strlist 中的位置

时间日期函数
日期获取
now()、curdate()、curtime()、utc_date()、utc_time(): 返回当前日期和时间、当前日期、当前时间、utc日期、utc时间
year(date)、month(date)、day(date): 返回日期参数的年份部分、月份部分、日部分
week(date): 返回日期参数的年份的第几周
weekday(date): 返回日期参数的星期索引（0 周一，1 周二，…，6 周日）
dayofyear(date): 返回日期参数的年中的天数

日期计算
adddate(date, INTERVAL expr unit): 返回日期加上一个时间间隔
addtime(date, expr): 返回日期加上一个时间
datediff(expr1, expr2): 返回两个日期之间的天数

日期格式化
date_format(date, format): 根据格式返回日期。
str_to_date(str, format): 根据格式返回日期。
format 可以是：%Y 年，%m 月，%d 日，%H 时，%i 分，%s 秒。

控制函数

单行函数
if(表达式,true_value,false_value)：if 函数处理有两个结果的流程
ifnull(字段名,null_value)：处理列等于 null 的时候,我们赋予其他的默认值
case: 处理多流程结果

聚合函数
avg(列名) 求平均值
sum(列名) 求和
min|max(列名) 求最小值和最大值
count(列名/*/1) 计数
注意：
聚合函数不能嵌套在其他函数中
聚合函数遇到null值，会自动忽略

mysql 高级查询

发表于 2025-02-04 分类于 mysql

分组查询

分组是将数据分成若干个组，并对每个组进行单独的查询，最终统计的是组的特性和数据（分组列，聚合函数）。
语法：
group by 分组列, 分组列, … hanving 分组后进行比较。
提示：
分组查询只能查询分组字段和聚合函数。
having是分组后的条件，where是分组前的条件。
having只能出现在group by后面，where可以出现在任何位置。
having通常比较的是聚合函数，where可以是任何参数比较。
having可以引用别名，where不可以，因为having是在分组之后执行，已经有了查询结果。

排序查询

语法：
order by 列名 asc/desc, 列名 asc/desc, …
提示：
asc升序 从小到大，desc降序 从大到小。
默认是asc，可以不写。
order by后面可以有多个列，按照第一个排序，如果相同则按第二个排序，以此类推。

分页查询

语法：
limit 起始索引, 每页条数
提示：
起始索引可以省略，省略默认为0。
limit影响的是查询结果，通常用在查询的最后。

查询的执行流程

关键字书写的顺序：
select -> from -> where -> group by -> having -> order by -> limit
关键字的执行顺序：
from -> where -> group by -> having -> select -> order by -> limit
提示：
mysql8.0+版本存在优化，使 group by 、having 可以使用 select 产生的列（别名）。

mysql 运算符

发表于 2025-02-04 更新于 2025-02-05 分类于 mysql

算数运算符

语法：
+（加） -（减） *（乘） /（除） div（整除） %（模）mod（模）
注意：
/ 浮点数除法，div整数除法。
/0 不会抛出异常，返回null。
浮点数类型运算结果为浮点数。
优先级：* / div % mod > + -，提升优先级使用括号。

比较运算符

比较对比：> < >= <=
等于对比：= 不能做空判断 null = null 结果为 false，null <=> null 结果为 true（mysql独有）
不等于对比：<> !=（mysql独有）
空值处理：is null is not null
区间比较：between min and max not between min and max
范围比较：key in(x,y,z) key = x or key = y or key = z not in
模糊匹配：like not like like 匹配规则，%任意字符出现0次或多次，_单一字符。

逻辑运算符

语法：and or not xor
注意：数据库中非空 非0代表true 反之为false, true -> 1 | false -> 0

运算符优先级

优先级 运算符 说明
1（最高） ! 逻辑非
2 *，/，div，%，mod 乘、除、整除、取模，取模
3 +，- 加、减
4 =，<=>，>=，>，<，<=，!=，IS，LIKE，IN，REGEXP 比较运算
5 BETWEEN，CASE，WHEN，THEN，ELSE 逻辑运算
6 NOT 逻辑非
7 AND 逻辑与
8 XOR 逻辑异或
9（最低） OR 逻辑或

注意：
一般情况下的运算符优先级。在实际使用中，如果不确定优先级，可以使用括号来明确运算的顺序！()内优先级最高！

mysql 字段类型

发表于 2025-02-03 更新于 2025-02-04 分类于 mysql

整数

类型 字节 有符号范围 无符号范围
INT 4 -2147483648 ~ 2147483647 0 ~ 4294967295
SMALLINT 2 -32768 ~ 32767 0 ~ 65535
TINYINT 1 -128 ~ 127 0 ~ 255
MEDIUMINT 3 -8388608 ~ 8388607 0 ~ 16777215
BIGINT 8 -263 ~ 263-1 0 ~ 264-1

列名 整数类型 -> 有符号 有负值和正值
列名 整数类型 UNSIGNED -> 无符号 只存正数, 负值部分用来存更大的正数

浮点数

类型 字节 M（小数位数+整数位数） D（小数位数）
FLOAT(M,D) 4 M最大位24 D最大为8
DOUBLE(M,D) 8 M最大位53 D最大为30
DECIMAL(M 动态计算 M最大位65 D最大为30

FLOAT, DOUBLE 类型在存储时会丢失精度，不适合用来做精确计算，如果要存储精确小数，可以使用decimal类型。
可以使用 UNSIGNED 属性修饰浮点类型，使其变为无符号。

字符串

类型 特点 长度范围（字符） 存储空间
CHAR(M) 固定长度 0 ~ 255 M*4个字节（utf8mb4）
VARCHAR(M) 可变长度 0 ~ *（一行数据最大占有空向是65535字节） M*4+1个字节（utf8mb4）

mysql中一行数据最大的占有空向是65535字节，所以varchar的长度不能超过65532字节，否则会报错。

文本类型

类型 描述 存储空间范围（字节）
TINYTEXT 小文本 0 ~ 255
TEXT 文本 0 ~ 65535
MEDIUMTEXT 中等文本 0 ~ 16777215
LONGTEXT 大文本 0 ~ 4294967295（4G）
文本类型不受mysql中一行数据最大的占有空向是65535字节的限制。

日期时间

类型 字节 格式 范围
YEAR 1 YYYY或YY 1901 ~ 2155 或 00~99
TIME 3 HH:MM:SS -838:59:59 ~ 838:59:59
DATE 3 YYYY-MM-DD 1000-01-01 ~ 9999-12-31
DATETIME 8 YYYY-MM-DD HH:MM:SS 1000-01-01 00:00:00 ~ 9999-12-31 23:59:59
TIMESTAMP 4 YYYY-MM-DD HH:MM:SS 1970-01-01 00:00:01 ~ 2038-01-19 03:14:07

扩展：DATETIME和TIMESTAMP类型可以设置自动初始化和更新
自动初始化：CREATE TABLE 表名 (字段名 DATETIME DEFAULT CURRENT_TIMESTAMP);
自动初始化，自动更新：CREATE TABLE 表名 (字段名 TIMESTAMP DEFAULT CURRETN_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP);

Dockerfile 用于自动化构建 Docker 镜像，定义镜像的环境、依赖和行为。
参考: https://docs.docker.com/reference/dockerfile/

示例

前端vue，后端nodejs的websocket服务，打包成docker镜像示例

前端镜像打包

1.本地vue项目构建打包出dist文件
2.编写nginx/nginx.conf文件
3.编写Dockerfile

4.将dist、nginx.conf、Dockerfile推送到服务器 /app/websocket/client 目录下
5.docker build -t websocketclient:v1.0 . (构建镜像，注意最后的点号表示当前目录为构建上下文)

1.编写Dockerfile
