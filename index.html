mysql 高级查询

发表于 2025-02-04 分类于 mysql

分组查询

分组是将数据分成若干个组，并对每个组进行单独的查询，最终统计的是组的特性和数据（分组列，聚合函数）。
语法：
group by 分组列, 分组列, … hanving 分组后进行比较。
提示：
分组查询只能查询分组字段和聚合函数。
having是分组后的条件，where是分组前的条件。
having只能出现在group by后面，where可以出现在任何位置。
having通常比较的是聚合函数，where可以是任何参数比较。
having可以引用别名，where不可以，因为having是在分组之后执行，已经有了查询结果。

排序查询

语法：
order by 列名 asc/desc, 列名 asc/desc, …
提示：
asc升序 从小到大，desc降序 从大到小。
默认是asc，可以不写。
order by后面可以有多个列，按照第一个排序，如果相同则按第二个排序，以此类推。

分页查询

语法：
limit 起始索引, 每页条数
提示：
起始索引可以省略，省略默认为0。
limit影响的是查询结果，通常用在查询的最后。

查询的执行流程

关键字书写的顺序：
select -> from -> where -> group by -> having -> order by -> limit
关键字的执行顺序：
from -> where -> group by -> having -> select -> order by -> limit
提示：
mysql8.0+版本存在优化，使 group by 、having 可以使用 select 产生的列（别名）。

算数运算符

语法：
+（加） -（减） *（乘） /（除） div（整除） %（模）mod（模）
注意：
/ 浮点数除法，div整数除法。
/0 不会抛出异常，返回null。
浮点数类型运算结果为浮点数。
优先级：* / div % mod > + -，提升优先级使用括号。

比较运算符

比较对比：> < >= <=
等于对比：= 不能做空判断 null = null 结果为 false，null <=> null 结果为 true（mysql独有）
不等于对比：<> !=（mysql独有）
空值处理：is null is not null
区间比较：between min and max not between min and max
范围比较：key in(x,y,z) key = x or key = y or key = z not in
模糊匹配：like not like like 匹配规则，%任意字符出现0次或多次，_单一字符。

逻辑运算符

语法：and or not xor
注意：数据库中非空 非0代表true 反之为false, true -> 1 | false -> 0

mysql 字段类型

发表于 2025-02-03 更新于 2025-02-04 分类于 mysql

整数

类型 字节 有符号范围 无符号范围
INT 4 -2147483648 ~ 2147483647 0 ~ 4294967295
SMALLINT 2 -32768 ~ 32767 0 ~ 65535
TINYINT 1 -128 ~ 127 0 ~ 255
MEDIUMINT 3 -8388608 ~ 8388607 0 ~ 16777215
BIGINT 8 -263 ~ 263-1 0 ~ 264-1

列名 整数类型 -> 有符号 有负值和正值
列名 整数类型 UNSIGNED -> 无符号 只存正数, 负值部分用来存更大的正数

浮点数

类型 字节 M（小数位数+整数位数） D（小数位数）
FLOAT(M,D) 4 M最大位24 D最大为8
DOUBLE(M,D) 8 M最大位53 D最大为30
DECIMAL(M 动态计算 M最大位65 D最大为30

FLOAT, DOUBLE 类型在存储时会丢失精度，不适合用来做精确计算，如果要存储精确小数，可以使用decimal类型。
可以使用 UNSIGNED 属性修饰浮点类型，使其变为无符号。

字符串

类型 特点 长度范围（字符） 存储空间
CHAR(M) 固定长度 0 ~ 255 M*4个字节（utf8mb4）
VARCHAR(M) 可变长度 0 ~ *（一行数据最大占有空向是65535字节） M*4+1个字节（utf8mb4）

mysql中一行数据最大的占有空向是65535字节，所以varchar的长度不能超过65532字节，否则会报错。

文本类型

类型 描述 存储空间范围（字节）
TINYTEXT 小文本 0 ~ 255
TEXT 文本 0 ~ 65535
MEDIUMTEXT 中等文本 0 ~ 16777215
LONGTEXT 大文本 0 ~ 4294967295（4G）
文本类型不受mysql中一行数据最大的占有空向是65535字节的限制。

日期时间

类型 字节 格式 范围
YEAR 1 YYYY或YY 1901 ~ 2155 或 00~99
TIME 3 HH:MM:SS -838:59:59 ~ 838:59:59
DATE 3 YYYY-MM-DD 1000-01-01 ~ 9999-12-31
DATETIME 8 YYYY-MM-DD HH:MM:SS 1000-01-01 00:00:00 ~ 9999-12-31 23:59:59
TIMESTAMP 4 YYYY-MM-DD HH:MM:SS 1970-01-01 00:00:01 ~ 2038-01-19 03:14:07

扩展：DATETIME和TIMESTAMP类型可以设置自动初始化和更新
自动初始化：CREATE TABLE 表名 (字段名 DATETIME DEFAULT CURRENT_TIMESTAMP);
自动初始化，自动更新：CREATE TABLE 表名 (字段名 TIMESTAMP DEFAULT CURRETN_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP);

参考: https://docs.docker.com/reference/dockerfile/

示例

前端vue，后端nodejs的websocket服务，打包成docker镜像示例

前端镜像打包

1.本地vue项目构建打包出dist文件
2.编写nginx/nginx.conf文件
3.编写Dockerfile

4.将dist、nginx.conf、Dockerfile推送到服务器 /app/websocket/client 目录下
5.docker build -t websocketclient:v1.0 . (构建镜像，注意最后的点号表示当前目录为构建上下文)

1.编写Dockerfile

docker 基础

发表于 2025-01-17 更新于 2025-01-18 分类于 docker

介绍

1.docker是一个容器技术，可以将应用打包成镜像，然后运行在容器中。
2.每个容器都是相互隔离的完整操作系统环境，互不影响。
3.通过端口映射、自定义网络，可以实现容器与容器与外部的通信。
4.通过卷挂载，可以实现容器内数据持久化。

存储

1.目录挂载

将本机的/app/nghtml目录挂载到容器内的/usr/share/nginx/html目录。

2.卷挂载

将本机的ngconfig卷挂载到容器内的/etc/nginx目录，如果卷不存在，则会自动创建，卷里默认数据是挂载容器目录的数据。
卷默认在本机的/var/lib/docker/volumes目录下。
修改卷数据，修改容器内数据，卷数据都会同步更新。
