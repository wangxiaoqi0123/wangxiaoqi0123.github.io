小程序基础使用

发表于 2021-02-27 更新于 2023-10-20 分类于 微信小程序

数据绑定

WXML中的动态数据均来自对应Page的data。

  • “Mustache”语法（双大括号）数据绑定
  • setData 添加/修改data中数据 数据绑定

生命周期钩子函数

  • 应用生命周期 onLaunch: 小程序初始化完成时触发，全局只触发一次。
    onShow: 小程序启动，或从后台进入前台显示时触发。
    onHide: 小程序从前台进入后台时触发。
    小程序生命周期文档

  • 页面生命周期 onLoad: 页面创建时执行。
    onShow: 页面出现在前台时执行。
    onReady: 页面首次渲染完毕时执行。
    onHide: 页面从前台变为后台时执行。
    onUnload: 页面销毁时执行。
    页面初始化时 onLoad > onShow > onReady
    页面生命周期文档

WXML中常用语法

  • 插值表达式{{}}
    放置 死值
    放置 变量
    可以支持运算符 > < ? : ( 三元运算符 ) + -

  • 属性
    属性={{死值 变量 运算符 > < ? : ( 三元运算符 ) + - * /}}

  • data-xxx 自定义属性 通过data-形式自定义的属性可以在event.currentTarget.dataset中获取到。
    1.自定义的属性在dataset中data-会被省略。例如 data-id => id
    2.当自定义属性中存在大写字母会被转换成小写。例如 data-postId => postid
    3.当自定义属性中存在多个-会被转换成驼峰。例如 data-post-id => postId

  • style 运算符写法
    style="{{color?'color:'+color+';':''}}{{size?'font-size:'+size+'rpx':''}}"

  • class 运算符写法
    class="text-class {{colorRed}} {{textSize}}"

  • wx:if 条件渲染 条件渲染

  • wx:for 列表渲染 wx:for-item 默认值 item
    wx:for-index 默认值 index
    列表渲染

  • 事件 bind 允许事件冒泡
    catch 阻止事件冒泡
    事件系统

路由跳转

  • navigateTo 保留当前页面，跳转到应用内的某个页面。
    不能跳到 tabbar 页面。
    不可以无限使用，小程序中页面栈最多十层。
    wx.navigateTo

  • redirectTo 关闭当前页面，跳转到应用内的某个页面。
    不能跳转到 tabbar 页面。
    wx.redirectTo

  • switchTab 跳转到 tabBar 页面，并关闭其他所有非 tabBar 页面
    wx.switchTab

  • 页面之间通讯 与浏览器地址传参一样，通过在url ？ 后面 以 key=value&key=value 的方式传递数据 在跳转后的页面通过onLoad生命周期函数中拿到传递的参数对象

数据缓存

  • App

    app.js文件中，在App内可以存储公共数据

    在小程序的任何位置都可以通过getApp()函数，进行访问和修改。
    当小程序重新加载时，数据会被初始化。

  • Storage

    小程序的Storage，类似于浏览器的localStorage，本地永久存储。

    setStorageSync 同步操作 增加/修改Storage数据。


    getStorageSync 同步操作 获取Storage中对应key的数据。


    removeStorageSync 同步操作 删除Storage中对应key的数据。


    clearStorageSync 同步操作 清空整个Storage数据。
