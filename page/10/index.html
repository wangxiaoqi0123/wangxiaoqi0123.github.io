JS数据变量内存

发表于 2020-07-14 更新于 2023-10-20 分类于 javascript

数据

  • 内存中存储的特定信息，本质上是一些0101…

内存

  • 可储存数据的空间
  • 一小块内存的两个部分
    • 内部存储的数据
    • 地址值
  • 内存分类
    • 栈: 全局变量/局部变量
    • 堆: 对象

变量

  • 可变化的量，由变量名和变量值组成
  • 每个变量都对应一小块内存，变量名用来查找对应的内存，变量值就是内存中保存的数据

内存数据变量三者之间的关系

  • 内存用来存储数据的空间
  • 变量是内存的标识

变量创建 赋值 读取 过程

  1. 创建 当声明 var obj = { name: ‘Tom’ }，相当于在堆内存开辟一块变量内存地址：0x123存储数据name:'Tom'，同时在栈内存开辟一块变量，变量名obj存储数据为堆内存的引用地址：0x123

  2. 赋值 当声明 var a = obj，相当于在栈内存开辟一块变量，变量名a，同时将栈内存中变量obj的数据拷贝一份，赋值给变量a

  3. 读取 当执行console.log(obj)，会根据栈内存中obj变量保存的堆内存的引用地址：0x123找到对应的堆内存，将其保存的值(数据)输出

Promise 介绍

发表于 2020-07-12 更新于 2023-10-20 分类于 javascript

1. Promise是一个构造函数

  • 一般把 一个功能 (或者请求) 封装成 一个函数 并将promise对象返回

2. Promise的链式操作

  • 封装好的函数返 return promise对象

  • then 的回调函数中 要把下一个要执行的函数 return

3. then 与 catch

  • then 回调的是resolve回调函数
  • catch 回调的是reject回调函数 , 且 一旦链式操作里面有 错误 则后面的then不会执行,直接执行catch

4. resolve 与 reject

  • Promise.resolve方法返回一个promise的实例，状态为 resolve

  • Promise.reject方法与 resolve 唯一的不同是，返回的 promise 对象的状态为 rejected

5. all

  • 所有的promise函数并行执行最后结果一起返回到 results

6. race

  • 谁先执行完成就返回谁的数据,其他的都不用

async await 使用方式

发表于 2020-07-12 更新于 2023-10-20 分类于 javascript

作用: 用于将 JS异步链式化 使代码从上向下顺序执行, 相当于promise的 .then 的链式操作

asyns

  1. 声明了此函数为异步函数
  2. 默认返回一个 promise 对象(函数后面可以接 .then 返回的结果就是 async函数里面 return的值 )

await

  1. 一般是一个 promise 封装的异步函数
  2. await的结果 是promise返回成功的回调 resolve 的值

Vue slot 插槽

1. 默认插槽

默认插槽 只有一个 添加后 组件标签对里面直接写入类容

2. 具名插槽

通过name 指定名称 添加后 组件标签对里 指定对应名称的slot 书写类容

3. 作用域插槽

通过 v-bind 传递出 数据 添加后 通过 v-slot:main=”scope” 下就可以接受组件传递出的数据

4. 默认值插槽

顾名思义 插槽内添加默认值
使用插槽时，插槽内部不添加任意值 就使用插槽 内默认的值

Vue 生命周期钩子

发表于 2020-07-11 更新于 2023-10-20 分类于 vue 2.0

  1. beforeCreate()
    可以获取this(组件 实例)，但无法获取真实的数据和节点

  2. created()
    可以获取this(组件 实例)和真实的数据，但无法获取真实的节点

  3. beforeMount()
    这个阶段，它渲染出了虚拟DOM

  4. mounted()
    这个阶段，它渲染出了真实DOM

  5. beforeUpdate()
    发生更新，可以获取更新后的数据，但是节点还未更新完

  6. updated()
    更新完成，数据和节点都是最新的

  7. beforeDestroy()
    组件销毁之前的一瞬间

  8. destroyed()
    组件被销毁

  9. activated()
    组件被激活

  10. deactivated()
    组件未激活

export与exports

发表于 2020-06-19 更新于 2023-10-20 分类于 javascript

CommonJS 方式

动态引入 代码执行时引入

CommonJS 解构导出

exports.key1 = value1
exports.key2 = value2
exports.key3 = value3

var {key1,key2,key3} = require(‘路径’)

CommonJS 对象导出

module.exports = { …}

var obj = require(‘路径’)

ES6 方式

静态引入，代码编译时引入

ES6 结构导出

export const key = value
export function fn () {}
export class Fn{}

import {key,fn,Fn} from “路径”

ES6 对象导出

export default {…}

import obj from “路径”

ES6 模块导入导出 只适合 结构的方式
export { key1 , key2} from “路径”
