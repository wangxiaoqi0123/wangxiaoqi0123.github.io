async await 使用方式

发表于 2020-07-12 更新于 2023-10-20 分类于 javascript

作用: 用于将 JS异步链式化 使代码从上向下顺序执行, 相当于promise的 .then 的链式操作

asyns

  1. 声明了此函数为异步函数
  2. 默认返回一个 promise 对象(函数后面可以接 .then 返回的结果就是 async函数里面 return的值 )

await

  1. 一般是一个 promise 封装的异步函数
  2. await的结果 是promise返回成功的回调 resolve 的值

Promise 介绍

发表于 2020-07-12 更新于 2023-10-20 分类于 javascript

1. Promise是一个构造函数

  • 一般把 一个功能 (或者请求) 封装成 一个函数 并将promise对象返回

2. Promise的链式操作

  • 封装好的函数返 return promise对象

  • then 的回调函数中 要把下一个要执行的函数 return

3. then 与 catch

  • then 回调的是resolve回调函数
  • catch 回调的是reject回调函数 , 且 一旦链式操作里面有 错误 则后面的then不会执行,直接执行catch

4. resolve 与 reject

  • Promise.resolve方法返回一个promise的实例，状态为 resolve

  • Promise.reject方法与 resolve 唯一的不同是，返回的 promise 对象的状态为 rejected

5. all

  • 所有的promise函数并行执行最后结果一起返回到 results

6. race

  • 谁先执行完成就返回谁的数据,其他的都不用

Vue 生命周期钩子

  1. beforeCreate()
    可以获取this(组件 实例)，但无法获取真实的数据和节点

  2. created()
    可以获取this(组件 实例)和真实的数据，但无法获取真实的节点

  3. beforeMount()
    这个阶段，它渲染出了虚拟DOM

  4. mounted()
    这个阶段，它渲染出了真实DOM

  5. beforeUpdate()
    发生更新，可以获取更新后的数据，但是节点还未更新完

  6. updated()
    更新完成，数据和节点都是最新的

  7. beforeDestroy()
    组件销毁之前的一瞬间

  8. destroyed()
    组件被销毁

  9. activated()
    组件被激活

  10. deactivated()
    组件未激活

Vue 父子组件通讯

发表于 2020-06-19 更新于 2023-10-20 分类于 vue 2.0

子组件向父组件传递消息

  1. 使用 ref 的方式
    (1) 父组件下 子组件标签 上标识 ref=”” 标识
    (2) 父组件 通过 this.$refs.标识 === 子组件Vue实例

  2. 父组件通过 $children 的方式获取子组件消息
    (1) 父组件 通过 this.$children === [子组件Vue实例 ,…] 子组件Vue实例的数组

  3. 子组件使用 $emit 的方法 发布订阅向父组件 传递消息
    (1) 子组件 发布 this.$emit(“订阅事件” , “要传递给父元素的数据”)
    (2) 父组件下 子组件标签 上添加监听事件 @订阅事件 = callback
    (3) 要接受的参数 在callback 中实现传递

父组件向子组件传递消息

  1. 使用 props 方式
    (1) 父组件下 子组件标签 通过数据属性向子组件传递数据 :数据名=”数据”
    (2) 子组件 同过 props:[“数据名”] 的方式接受数据
  2. 使用 $parent 的方式获取父元素上的信息
    (1) 子组件 通过 this.$parent == 父组件Vue实例 (父组件唯一所以不是数组)

兄弟组件之间传递消息(非父子组件)

  1. 只有通过 Vue实例 实现通讯的一种方式
    (1) 创建一个 Vue实例 专门用于实现兄弟元素之间的通讯 var bus = new Vue()
    (2) 发送消息的兄弟 在 methods 中 使用 bus.$emit(“名称” , 数据)
    (3) 接受消息的兄弟 在 mounted (生命周期回调函数) 使用 bus.$on(“名称”,callback) 回调函数中接受消息

watch监听属性 computed计算属性

发表于 2020-06-19 更新于 2023-10-20 分类于 vue 2.0

computed
是计算值 有get set 两个方法 默认是 get
应用：就是简化tempalte里面计算和处理props或$emit的传值
具有缓存性，页面重新渲染值不变化,计算属性会立即返回之前的计算结果，而不必再次执行函数

watch
是观察的动作，
应用：监听props，$emit或本组件的值执行异步操作
无缓存性，页面重新渲染时值不变化也会执行

用法

  • 相同

    1. computed 与 watch 里面计算/监听的函数 形式上是函数 实际上是变量

  • 区别一

    1. copmuted 计算的是一个自定义的变量 变量不可以在 data 中存在
    2. watch 监听的是一个必须 已经存在的变量 (data , computed, props …)

  • 区别二

    1. computed 必须依赖一个或者多个数据项,然后对数据进行操作计算, 从而返回一个计算值 ,有get set 两个方法 默认是get方法, set方法 手动添加 可以接受一个修改后的值 val 形参
    2. watch 监听的是一个存在的变量, 且 函数可以接受 newVal, oldVal 两个形参

  • 区别三

    1. computed 不能够实现异步操作
    2. watch 允许异步操作

computed 使用

  1. 默认get方法 直接依赖data props computed 等等 中数据

  2. get set 写法

watch 使用

  1. 默认写法 直接监听data中数据的变化
  2. 对象深度监听
  3. immediate 数据首次绑定watch立即执行

总结

computed return 依赖值计算的结果
watch 监听的变量 发生变化时执行函数内部的语句

export与exports

发表于 2020-06-19 更新于 2023-10-20 分类于 javascript

CommonJS 方式

动态引入 代码执行时引入

CommonJS 解构导出

exports.key1 = value1
exports.key2 = value2
exports.key3 = value3

var {key1,key2,key3} = require(‘路径’)

CommonJS 对象导出

module.exports = { …}

var obj = require(‘路径’)

ES6 方式

静态引入，代码编译时引入

ES6 结构导出

export const key = value
export function fn () {}
export class Fn{}

import {key,fn,Fn} from “路径”

ES6 对象导出

export default {…}

import obj from “路径”

ES6 模块导入导出 只适合 结构的方式
export { key1 , key2} from “路径”
