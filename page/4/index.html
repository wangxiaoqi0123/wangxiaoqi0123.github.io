JS原型链

发表于 2020-07-15 更新于 2020-07-17 分类于 javascript

原型链

  • 访问一个对象属性时
    • 先在自身属性中查找，找到返回
    • 如果没有，再沿着__proto__这条链先上查找，找到返回
    • 如果最终没找到，返回undefined
  • 作用：查找对象的属性（方法）

构造函数/原型/实例对象的关系（图解）

  • 声明构造函数时 会添加prototype属性 它的值是一个Object空对象
  • Object空对象为构造函数Object的实例，它的隐式原型__prpto__指向构造函数的显式原型
  • 构造函数Object的原型对象的隐式原型__proto__的值为 null

Function/Object/自定义构造函数（图解）

  • 构造函数Function通过new自身创建，他的显式原型与隐式原型是同一个对象
  • 构造函数Object与自定义构造函数通过new Function 创建，属于构造函数Function的实例，它们的隐式原型__proto__指向构造函数Function的显式原型对象。

原型链继承

  1. 定义父类型构造函数
  2. 给父类型的原型添加方法
  3. 定义子类型构造函数
  4. 把父类型的实例对象赋值给子类型的原型
  5. 给子类型的原型对象上添加方法
  6. 实例子类型对象：可以调用父类型的方法

JS函数的prototype

发表于 2020-07-15 更新于 2020-07-16 分类于 javascript

函数的prototype属性

  • 每个函数都有一个prototype属性，它默认指向一个Object空对象（即称为：原型对象）
  • 原型对象中有一个属性 constructor，它指向函数对象
  • 给原型对象添加属性（一般都是方法），函数所有的实例对象自动拥有构造函数原型中的属性（方法）

显式原型与隐式原型

  • 每个函数function都有一个 prototype，即显式原型（属性）

  • 每个实例对象都有一个__proto__，可称为隐式原型（属性）

  • 对象的隐式原型的值为其对应构造函数的显示原型的值

  • 函数的prototype属性：在定义函数时自动添加，默认值是一个空的Object对象

  • 对象的__proto__属性：创建对象时自动添加，默认值是构造函数的prototype属性值

JS数据变量内存

发表于 2020-07-14 更新于 2020-07-16 分类于 javascript

数据

  • 内存中存储的特定信息，本质上是一些0101…

内存

  • 可储存数据的空间
  • 一小块内存的两个部分
    • 内部存储的数据
    • 地址值
  • 内存分类
    • 栈: 全局变量/局部变量
    • 堆: 对象

变量

  • 可变化的量，由变量名和变量值组成
  • 每个变量都对应一小块内存，变量名用来查找对应的内存，变量值就是内存中保存的数据

内存数据变量三者之间的关系

  • 内存用来存储数据的空间
  • 变量是内存的标识

变量创建 赋值 读取 过程

  1. 创建 当声明 var obj = { name: ‘Tom’ }，相当于在堆内存开辟一块变量内存地址：0x123存储数据name:'Tom'，同时在栈内存开辟一块变量，变量名obj存储数据为堆内存的引用地址：0x123

  2. 赋值 当声明 var a = obj，相当于在栈内存开辟一块变量，变量名a，同时将栈内存中变量obj的数据拷贝一份，赋值给变量a

  3. 读取 当执行console.log(obj)，会根据栈内存中obj变量保存的堆内存的引用地址：0x123找到对应的堆内存，将其保存的值(数据)输出

JS数据类型

发表于 2020-07-14 更新于 2020-07-16 分类于 javascript

分类

  • 基本（值）类型
    • String: 任意字符串
    • Number: 任意数字
    • Boolean: true/false
    • undefined: undefined
    • null: null
  • 对象（引用）类型
    • Object: 任意对象
    • Function: 一种特别的对象（可书写逻辑，可执行）
    • Array: 一种特别的对象（数值下标，内部数据是有序的）

判断

  • typeof: 返回数据类型的字符串表达
    • 可以判断: undefined / 数值 / 字符串 / 布尔值 / function
    • 不能判断: null与object object与array
  • instanceof
    • 判断对象的具体类型（实例 instanceof 构造函数）true/false
  • ===
    • 可以判断：undefined , null

undefined

  • undefined代表变量定义未赋值
  • null定义并赋值了，只是值为null

null

  • 初始赋值，表明将要赋值为对象
  • 让对象成为垃圾对象（被垃圾回收机制回收）

严格区分变量类型与数据类型

  • 数据的类型
    • 基本类型
    • 对象类型
  • 变量的类型（变量内存值得类型）
    • 基本类型: 保存的为基本类型的数据
    • 引用类型: 保存的为内存引用地址

asyns

  2. 默认返回一个 promise 对象(函数后面可以接 .then 返回的结果就是 async函数里面 return的值 )

await

  2. await的结果 是promise返回成功的回调 resolve 的值

Promise 介绍

发表于 2020-07-12 更新于 2020-09-05 分类于 javascript

1. Promise是一个构造函数

  • 一般把 一个功能 (或者请求) 封装成 一个函数 并将promise对象返回

2. Promise的链式操作

  • 封装好的函数返 return promise对象

  • then 的回调函数中 要把下一个要执行的函数 return

3. then 与 catch

  • then 回调的是resolve回调函数
  • catch 回调的是reject回调函数 , 且 一旦链式操作里面有 错误 则后面的then不会执行,直接执行catch

4. resolve 与 reject

  • Promise.resolve方法返回一个promise的实例，状态为 resolve

  • Promise.reject方法与 resolve 唯一的不同是，返回的 promise 对象的状态为 rejected

5. all

  • 所有的promise函数并行执行最后结果一起返回到 results

6. race

  • 谁先执行完成就返回谁的数据,其他的都不用

Vue 生命周期钩子

发表于 2020-07-11 更新于 2020-07-16 分类于 vue 2.0

  1. beforeCreate()
    可以获取this(组件 实例)，但无法获取真实的数据和节点

  2. created()
    可以获取this(组件 实例)和真实的数据，但无法获取真实的节点

  3. beforeMount()
    这个阶段，它渲染出了虚拟DOM

  4. mounted()
    这个阶段，它渲染出了真实DOM

  5. beforeUpdate()
    发生更新，可以获取更新后的数据，但是节点还未更新完

  6. updated()
    更新完成，数据和节点都是最新的

  7. beforeDestroy()
    组件销毁之前的一瞬间

  8. destroyed()
    组件被销毁

  9. activated()
    组件被激活

  10. deactivated()
    组件未激活

  2. 复制 Font class 在线生成 的地址

  1. 标签 clas 添加 iconfont 和 对应图标名

  5. 当项目 添加新的 图标 整个文件夹需要重新打包 替换
