vue自定义指令directive

发表于 2020-09-05 更新于 2023-10-20 分类于 vue 2.0

简单介绍

  1. vue官方介绍：对普通 DOM 元素进行底层操作，这时候就会用到自定义指令。
  2. 所以我认为 directive 的绝大多数用于处理，静态的UI交互逻辑的封装。

与 filter component mixin 注册使用方式类似，directive 也是使用 install function 在这种方式注册

  1. 局部注册

    • 书写指令
    • 引入组件局部注册

  2. 全局注册

    • 书写指令
    • Vue.use 全局注册
    • main.js 导入

常用钩子函数介绍

指令的钩子函数 一共5个 bind inserted update componentUpdated unbind 具体详细解释看官网，这里只介绍常用的两个，以及常用的钩子函数参数。

钩子函数
  1. bind
    • 只调用一次，指令第一次绑定到元素时调用
  1. update
    • 官方：所在组件的 VNode 更新时调用，但是可能发生在其子 VNode 更新之前。
    • 简单理解：当指令节点下的视图发生变化时，触发update。
钩子函数参数
  1. el 指令所绑定的元素，可以用来直接操作 DOM。
  1. binding 一个对象，包含以下常用 property
    • value: 指令传入的绑定值。
    • oldValue: 指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用。

  2. 父组件部分：父组价通过自定义属性把数据传递出去，再监听自定义事件，根据自定义事件传递的数据修改属性值

  1. 子组件部分

  2. 父组件部分

语法糖解析

通过以上的实现方式，结合vue官网，我们就可以清楚的知道，当我们使用 :text.sync 实际完成了一下几个事情

  1. 将自定义属性text通过props传递给子组件
  2. 监听v-on:update:text 事件触发
  3. v-on:update:text事件触发会传递过来的数据复制给当前text

Decorator装饰器

发表于 2020-08-11 更新于 2023-10-20 分类于 javascript

装饰器介绍

  1. 装饰器（Decorator）是ES2016提出来的一个提案，当前处于Stage 2阶段，目前还没有定案。
  2. 本质上是一个函数，使用方式以 @ + 函数名 的形式。
  3. 是一种与类（class）相关的语法，用来包装或者修改类或者类的方法的行为。

装饰器属性描述

  1. target
    • 对应修饰方法的父类
  2. name
    • 对应修饰方法的名称(string)
  3. descriptor
    • descriptor.configurable：表示能不能通过delete删除属性，能否修改属性的其他描述符特性，或者能否将数据属性修改为访问器属性。当我们通过let obj = {name: ‘’}声明一个对象的时候，这个对象里面所有的属性的configurable描述符的值都是true
    • descriptor.enumerable：表示能不能通过for in或者Object.keys等方式获取到属性，我们一般声明的对象里面这个描述符的值是true,但是对于class类里面的属性来说，这个值是false
    • descriptor.writable：表示能否修改属性的数据值，通过将这个修改为false,可以实现属性只读的效果。
    • descriptor.value：表示当前属性的数据值，读取属性值的时候，从这里读取；写入属性值的时候，会写到这个位置。

类的装饰

  • 类的装饰 一般主要使用 target 属性在其类上添加属性与方法

方法的装饰

  • 方法的装饰 一般组要使用 descriptor.value 属性对当前修饰的方法进行重写 包装
  • 一般会包裹一层方法用于传递参数

移动端rem适配

简单实现rem适配

  1. 使用onresize事件，当设备窗口发生变化时，动态设置根节点的font-size
  2. 设置尺寸时，以标准设计稿为基础设置，一般以750视口下font-size为100px动态换算当前设备的font-size（750一般为设计稿尺寸）

淘宝适配与网易适配

  1. 网易适配: 设计稿宽度是 750px，假定当前设计稿的 1rem = 100px，那么 屏幕宽度：设计稿宽度 = 设备的rem:设计稿假定的rem(100)
  2. 淘宝适配: 把设备宽度 100vw 分成 10等份，假设当前设备的rem 等于 其中的一份，那么1rem = 10vw

lib-flexible与postcss-px2rem-exclude适配方案

  1. 设置理想视口：把默认的layout viewport的宽度设为移动设备的屏幕宽度，得到理想视口
  2. 下载安装
  3. mian.js引入 lib-flexible
  4. 创建.postcssrc.js 使用postcss-px2rem-exclude将px转译为rem 或者

postcss-px-to-viewport适配方案

  1. 设置理想视口：把默认的layout viewport的宽度设为移动设备的屏幕宽度，得到理想视口
  2. 下载安装
  3. 创建.postcssrc.js 使用postcss-px-to-viewport将px转译为vw
  4. 查看postcss-px-to-viewport文档

JS防抖与节流

发表于 2020-08-03 更新于 2023-10-20 分类于 javascript

防抖（debounce）

触发事件后在规定时间内函数只能执行一次，如果在规定时间内又触发了事件，则重新计算函数执行时间

  1. 简单实现
  2. 原理
    1. 利用闭包延长 timeout 变量生命周期
    2. 除了第一次函数执行外，每执行一次函数，先关闭定时器，再重新打开定时器

节流（throttle）

事件连续触发时，在规定时间内只执行一次函数

  1. 简单实现
  2. 原理
    1. 利用闭包延长 timeout 变量生命周期
    2. 执行函数时，当 timeout 存在值时不执行定时器，定时器执行完成时将 timeout 置为 null

项目中使用

  • 函数防抖和节流
    1. 在公共方法中（如 utils.js 中），加入函数防抖和节流方法
    2. 在需要使用的组件引用
    3. 在 methods 中使用
  • 直接使用npm 安装（指令）
    • https://www.npmjs.com/package/v-debounce-throttle
  • 项目中直接引入文件手动 use 安装 （指令）
    • https://github.com/wangxiaoqi0123/vue-cli3-vant/tree/master/src/directive/debounce-throttle

Vue $attrs与$listeners

发表于 2020-07-22 更新于 2023-10-20 分类于 vue 2.0

简单讲

  1. $attrs 将父组件上的自定义属性，传递到指定的子属性的内部
  2. $listeners 子组件上触发的事件，在父组件上也可以监听到

$attrs

  1. 将父组件上的自定义属性，通过v-bind='$attrs'，传递到绑定的子组件内部
  2. 不包含所有父作用域的绑定 class 和 style
  3. inheritAttrs默认为true，通过设置 inheritAttrs: false 取消组件的根元素也同样继承属性

  1. 父组件在页面上使用

  2. 父组件内给子组件设置 v-bind:'$attrs'

  3. 子组件内部接受数据 (props/$atters)

$listeners

  1. 将子组件内触发的事件，传递到父组件上
  2. 通过在父组件内部 给子组件 设置 v-on="$listeners"，父组件标签上就可以自定义子组件内触发的事件

  1. 父组件在页面上使用

  2. 父组件内给子组件设置 v-on="$listeners"

  3. 子组件内发布是事件
