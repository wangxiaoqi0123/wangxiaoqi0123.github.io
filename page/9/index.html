Promise 介绍

发表于 2020-07-12 更新于 2023-10-20 分类于 javascript

1. Promise是一个构造函数

  • 一般把 一个功能 (或者请求) 封装成 一个函数 并将promise对象返回

2. Promise的链式操作

  • 封装好的函数返 return promise对象

  • then 的回调函数中 要把下一个要执行的函数 return

3. then 与 catch

  • then 回调的是resolve回调函数
  • catch 回调的是reject回调函数 , 且 一旦链式操作里面有 错误 则后面的then不会执行,直接执行catch

4. resolve 与 reject

  • Promise.resolve方法返回一个promise的实例，状态为 resolve

  • Promise.reject方法与 resolve 唯一的不同是，返回的 promise 对象的状态为 rejected

5. all

  • 所有的promise函数并行执行最后结果一起返回到 results

6. race

  • 谁先执行完成就返回谁的数据,其他的都不用

async await 使用方式

发表于 2020-07-12 更新于 2023-10-20 分类于 javascript

作用: 用于将 JS异步链式化 使代码从上向下顺序执行, 相当于promise的 .then 的链式操作

asyns

  1. 声明了此函数为异步函数
  2. 默认返回一个 promise 对象(函数后面可以接 .then 返回的结果就是 async函数里面 return的值 )

await

  1. 一般是一个 promise 封装的异步函数
  2. await的结果 是promise返回成功的回调 resolve 的值

watch监听属性 computed计算属性

发表于 2020-06-19 更新于 2023-10-20 分类于 vue 2.0

computed
是计算值 有get set 两个方法 默认是 get
应用：就是简化tempalte里面计算和处理props或$emit的传值
具有缓存性，页面重新渲染值不变化,计算属性会立即返回之前的计算结果，而不必再次执行函数

watch
是观察的动作，
应用：监听props，$emit或本组件的值执行异步操作
无缓存性，页面重新渲染时值不变化也会执行

用法

  • 相同

    1. computed 与 watch 里面计算/监听的函数 形式上是函数 实际上是变量

  • 区别一

    1. copmuted 计算的是一个自定义的变量 变量不可以在 data 中存在
    2. watch 监听的是一个必须 已经存在的变量 (data , computed, props …)

  • 区别二

    1. computed 必须依赖一个或者多个数据项,然后对数据进行操作计算, 从而返回一个计算值 ,有get set 两个方法 默认是get方法, set方法 手动添加 可以接受一个修改后的值 val 形参
    2. watch 监听的是一个存在的变量, 且 函数可以接受 newVal, oldVal 两个形参

  • 区别三

    1. computed 不能够实现异步操作
    2. watch 允许异步操作

computed 使用

  1. 默认get方法 直接依赖data props computed 等等 中数据

  2. get set 写法

watch 使用

  1. 默认写法 直接监听data中数据的变化
  2. 对象深度监听
  3. immediate 数据首次绑定watch立即执行

总结

computed return 依赖值计算的结果
watch 监听的变量 发生变化时执行函数内部的语句

Vue 路由跳转 router

发表于 2020-06-19 更新于 2023-10-20 分类于 vue 2.0

Vue 页面中实现跳转

  1. this.$router.push()
    • this.$router.push(‘路由路径’)
    • this.$router.push({name:’组件名称’,path:’路由路径’})
    • push 方式跳转 导航到不同的 url，向 history 栈添加一个新的记录
  2. this.$router.replace()
    • this.$router.replace(‘路由路径’)
    • this.$router.replace({name:’组件名称’,path:’路由路径’})
    • replace 方式跳转 导航到不同 url，替换 history 栈中当前记录
  3. this.$router.go()
    • router.go(n)：指定前进/回退的步数。n 为正数的时候是前进；负数的时候是后退；0的时候是刷新当前页面。（===window.history.go）
  4. router.back()
    • 回退一步
  5. router-link

页面的路由信息对象
this.$route

Vue 页面中跳转（push/replace） 带参数

  1. 跳转传参
  • 方式一 使用params 传递参数
    this.$router.push({path:’组件路径/‘,params:{变量名:值}})
    this.$router.push({name:’组件名称’,params:{变量名:值}})
  • 方式二 使用query 传递参数
    this.$router.push({path:’组件路径/‘,query:{变量名:值}})
    this.$router.push({name:’组件名称’,query:{变量名:值}})
  1. 接受参数
  • this.$route.params.变量名 (一般在 mounted 中接受参数) 页面路由看不见 页面刷新参数消失
  • this.$route.query.变量名 (一般在 mounted 中接受参数) 页面路由显示 页面刷新参数不消失

模块(组件)上的 v-model 实现

发表于 2020-06-19 更新于 2023-10-20 分类于 vue 2.0

实现理论

  1. v-model 实际上是 @input + :value 的语法糖
  2. vue 中的 model 允许自定义里面 prop(组件 v-mode 值) event(事件类型)
  3. 两个方向数据双向绑定
    父向同过 v-model 向组件传值
    组件通过 this.$emit(“input”, value) 向父传值

注意事项

页面通过 自定义属性的方式组件传值 涉及到异步数据 要对数据进行监听!!!

常用两种实现方式

  1. watch监听 和 过渡属性 immediate: true

  2. 计算属性 setter getter(建议使用)

  2. 复制 Font class 在线生成 的地址
